添加普通用户

使用root用户登录Linux服务器，并创建普通用户

一般来说， 一个项目会由多个开发人员协作完成 。 为了节省企业成本，公司不会为每个开发人员都配备一台服务器 ， 而是让所有的开发人员共用一个开发机，并以普通用户的身份登录开发机进行开发。

尽量避免使用root账户 ， 以普通用户登录和操作开发机可以保证系统的安全性

命令

useradd pying  # 创建用户
passwd pying    # 设置密码

演示

[root@localhost ~]# useradd pying
[root@localhost ~]# passwd pying

添加sudoers

命令

# 把用户pying添加/etc/sudoers文件
sed -i '/^root.*ALL=(ALL)/a\pying\tALL=(ALL) \tALL'  /etc/sudoers
# 在/etc/sudoers文件中查找
# root	ALL=(ALL) 	ALL
# 在此行下添加
# pying	ALL=(ALL) 	ALL

或者

# 把用户pying添加wheel组
usermod -aG wheel pying
# -a 添加用户到基本的组，仅和 -G选项一起使用
# -G 用户也是其成员的补充组列表。 每个组之间用逗号分隔，中间没有空格。如果用户当前是未列出的组的成员，则该用户将从该组中删除。

测试

sudo -i

替换CentOS8.4系统自带的Yum源

官方的Yum源已不可用

阿里镜像源 https://mirrors.aliyun.com/repo/

命令

mv /etc/yum.repos.d/ /etc/yum.repos.d.bak
mkdir -p /etc/yum.repos.d/
wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-vault-8.5.2111.repo
dnf clean all && dnf makecache

配置$HOME/.bashrc文件

使用pying用户登录

配置$HOME/.bashrc文件，使Linux Shell更加易用

vim $HOME/.bashrc
原文件内容：

# .bashrc
# Source global definitions
if [ -f /etc/bashrc ]; then
        . /etc/bashrc
fi

# User specific environment
if ! [[ "$PATH" =~ "$HOME/.local/bin:$HOME/bin:" ]]
then
    PATH="$HOME/.local/bin:$HOME/bin:$PATH"
fi
export PATH

配置后内容如下：

# .bashrc
# Source global definitions
if [ -f /etc/bashrc ]; then
        . /etc/bashrc
fi

# User specific environment
if ! [[ "$PATH" =~ "$HOME/.local/bin:$HOME/bin:" ]]
then
    PATH="$HOME/.local/bin:$HOME/bin:$PATH"
fi
export PATH

# 用户特定的别名函数
# 操作前询问
alias rm ='rm -i'
alias cp ='cp -i'
alias mv ='mv -i'

# 源全局定义
# 判断/etc/bashrc这个文件是否存在，如果存在执行这个文件
#if [ -f /etc/bashrc ]; then
#    . /etc/bashrc
#fi
# 判断$HOME/workspace是否为目录,不是则创建
if [ ! -d $HOME/workspace ]; then
    mkdir -p $HOME/workspace
fi

# 用户特定环境
# Basic envs

export LANG="en_US.UTF-8"   # 设置系统语言为en_US.UTF-8 避免中断出现中文乱码
export PS1='[\u@dev \w]\$ '    # 默认的PS1设置会展示全部的路径，为防止过长，这里只展示: "用户名 @dev 最后的目录名"
export WORKSPACE="$HOME/workspace"  # 设置工作目录

export PATH=$HOME/bin:$PATH # 将$HOME/bin目录加入PATH变量中

# 设置默认入口目录
cd $WORKSPACE   # 登录系统 默认进入workspace目录

配置好后

bash

注意

export PATH 时，最好把$PATH放在最后，一维添加到$HOME/bin目录中的命令是期望被优先搜索并使用的

$HOME/.bashrc会自动创建目录workspace，所有的工作都在这个目录下展开:

  • 可以使$HOME保持整洁
  • 分区空间不足，可以移动workspace到另一个分区，并在分区中保留软连接，如：/home/pying/workspace -> /data/workspace/
  • 备份所有工作文件，直接备份workspace

依赖安装和配置

安装依赖

dnf install -y make cmake autoconf automake cmake perl-CPAN libcurl-devel libtool gcc gcc-c++ glibc-headers zlib-devel git-lfs telnet lrzsz jq expat-devel openssl-devel bzip2-devel ncurses-devel sqlite-devel readline-devel tk-devel libffi-devel

安装Git

cd /tmp
wget --no-check-certificate https://mirrors.edge.kernel.org/pub/software/scm/git/git-2.39.2.tar.gz
tar -zxvf git-2.39.2.tar.gz
cd git-2.39.2
./configure
make
sudo make install

# 将Git二进制目录添加到 PATH 路径中 
tee -a $HOME/.bashrc <<'EOF'
export PATH=/usr/local/libexec/git-core:$PATH
EOF

配置Git

# 修改用户名
git config --global user.name "gzwillyy"
# 修改邮箱
git config --global user.email "gzwillyy@gmail.com"
# 设置git，保存用户名和密码
git config --global credential.helper store 
# 解决git中 'Filename too lang' 的错误
git config --global core.longpaths true 
# 防止乱码
git config --global core.quotepath off
# github限制最大克隆 需要安装 Git Large File Storage
git lfs install --skip-repo
# 设置自动更正
git config --global help.autocorrect 15
# Git 中设置 GPG 签名主键,替换要使用的 GPG 密钥ID： 3AA5C34371567BD2
git config --global user.signingkey 3AA5C34371567BD2
# 将 Git 配置为默认对所有提交进行签名
git config --global commit.gpgsign true
