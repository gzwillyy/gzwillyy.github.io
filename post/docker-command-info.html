批量操作命令

列出所有 container的 ID

docker container ps -aq

批量停止运行所有 container

docker container stop $(docker container ps -aq)

批量删除所有 container

docker container rm $(docker container ps -aq)

强制删除运行中的 container

docker container rm $(docker container ps -aq) -f

运行停止后自动删除的 container

docker container run --rm -it  <Image ID>

清除所有停止的 container

docker system prune -f

清除所有未使用的 Image镜像

docker image prune -a

容器 attached 和 detached 运行模式

attached 模式在前台运行，ctrl + c 会终止运行，信号传递到容器内部
detached 模式在后台运行

# attached
docker container run -p 80:80 nginx

# detached
docker container run -d -p 80:80 nginx

# 重新 attach 到容器
docker container attach <Container ID>

# 或者查看日志
docker container logs <Container ID>
# 动态跟踪日志
docker container logs <Container ID> -f

容器 交互式模式 -it

# 交互式创建运行容器并进入 ubuntu 容器 shell 中
# exit 退出 shell 脚本环境， 容器会停止运行
docker container run -it ubuntu sh

# 已经运行的容器通过命令进入交互式模式
# exit 退出 shell 脚本环境， 容器不会停止运行
docker exec -it <Container ID> sh

交互式运行 busybox

docker container run -it busybox sh
# container 的ip地址
ip a
# ping
ping www.baidu.com

Busybox 是一个精简的 Linux 工具集合，你可以使用它进行文件操作、查看网络配置、测试网络连通性等。它提供了许多常用的命令和工具，包括文件操作、文本处理、网络工具等等。

Busybox 镜像的主要用途是在 Docker 容器中提供一个轻量级的基础环境。

它可以用作其他 Docker 镜像的基础镜像，也可以作为临时容器用于调试和测试。

由于 Busybox 镜像非常小巧，仅包含最基本的工具和命令，因此它可以帮助减小容器的大小和资源占用，并提高容器的启动速度。

它适用于那些只需要基本功能的应用场景，或者在构建定制化的容器镜像时作为基础层使用。

Dockerfile 介绍

https://docs.docker.com/engine/reference/builder/

  • Dockerfile 是用于构建docker镜像的文件

  • Dockerfile 里包含了构建镜像所需的“指令”

  • Dockerfile 有其特定的语法规则

举例：执行一个Python程序

容器即进程，所以镜像就是一个运行这个进程所需要的环境。

假如我们要在一台ubuntu 22.04上运行下面这个 hello.py 的 Python 程序

hello.py的文件内容：

第一步，准备Python环境

apt-get update && \
DEBIAN_FRONTEND=noninteractive apt-get install --no-install-recommends -y python3.10 python3-pip python3.10-dev

第二步，运行 hello.py

$ python3 hello.py
hello docker

一个Dockerfile的基本结构

Dockerfile

FROM ubuntu:22.04
RUN apt-get update && \
    DEBIAN_FRONTEND=noninteractive apt-get install --no-install-recommends -y python3.10 python3-pip python3.10-dev
ADD hello.py /
CMD ["python3", "/hello.py"]

镜像的构建和分享

从 Dockerfile 构建镜像

# . 指定 dockerfile 文件位置
docker image build -t hello:1.0.0 .

# 运行s
docker run -it hello:1.0.0

将镜像 push 到 dockerhub


# 从新创建镜像
docker image build -t <账户>/hello:1.0.0 .

# 或者 使用 docker image tag 根据已经存在的 image 创建一个新的tag image
docker image tag <Image ID>  <账户>/hello:1.0.1

# 登录
docker login <地址> -u <账户> -p <密码>

# 推送到远程仓库
docker image push <账户>/hello:1.0.1

通过 commit 创建镜像

从 Container commit 一个 Image 镜像

# 运行一个容器
docker container run -d -p 80:80 nginx
# 进入容器内部
docker container exec -it <Container ID> sh
# 编辑修改容器里内容
cd /usr/share/nginx/html
echo "<h1>hello<h1>" > index.html

# exit
# 停止容器
docker container stop <Container ID>
# 启动容器 查看修改
docker container start <Container ID>

# 帮助命令
docker container commit --help

# 创建镜像
docker container commit <Container ID> <账户>/nginx:<Tag>

# 查看创建的镜像 Image
docker images

关于 scratch 镜像

Scratch 是一个空的 Docker 镜像。

通过 scratch 来构建一个基础镜像。

hello.c

#include <stdio.h>
int main()
{
    printf("hello docker\n");
}

编译成一个二进制文件

gcc --static -o hello hello.c
./hello
hello docker

Dockerfile

FROM scratch
ADD hello /
CMD ["/hello"]

构建

docker build -t hello .
docker image ls
REPOSITORY   TAG       IMAGE ID       CREATED          SIZE
hello        latest    2936e77a9daa   40 minutes ago   872kB

运行

docker container run -it hello
hello docker

查看分层

docker image history hello
